Memory Architecture
Every session reads CLAUDE.md first. Threads load only the memory files relevant to their task.
.claude/
├── CLAUDE.md ← Thin router (~30 lines). Points to memory/ files.
├── memory/
│ ├── identity.md ← Who Scott is, 5 businesses, finances, properties, accounts
│ ├── people.md ← Contact glossary. All threads can append new contacts.
│ ├── working-rules.md ← Operating rules, comm preferences, output defaults
│ ├── active-context.md ← Live project statuses, pending decisions, key dates
│ ├── thread-map.md ← Thread inventory, scheduled tasks, data flow, connectors
│ ├── projects/ ← Deep-dive files per deal (Patron, Pioneer, Cresset)
│ └── context/ ← Balance sheet, insurance details, reference data
├── tasks/
│ └── lessons.md ← Mistake log + operating patterns
└── skills/
├── travel-research-agent/ ← P2a travel skill + preferences
├── badge-tracking-ups-report/ ← NSH badge tracking
├── masters-price-monitor/ ← StubHub/Vivid scraper
├── crawford-marketing-agent/ ← Campaign segmentation, RFM scoring
├── crawford-restaurant-site/ ← Website builder/deployer
├── crawford-site-audit/ ← SEO + strategy deck generator
├── wine-allocation-tracker/ ← Portfolio + secondary market
└── prototype-designer/ ← Competitor research → HTML prototypes
Update Protocol
New Contact Discovered
Append to memory/people.md with name, role, contact info, and notes.
Project Status Changed
Update memory/active-context.md — statuses, dates, blockers.
Mistake Made
Log to tasks/lessons.md — what happened, pattern, prevention.
New Thread Created
Update memory/thread-map.md — add thread, domain, status, memory dependencies.
